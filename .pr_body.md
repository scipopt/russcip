
## Motivation

`create_sol` / `create_orig_sol` produce **complete** solutions: every variable
left unset defaults to `0`, and the assignment must be feasible to be accepted.
That makes it impossible to give a *partial* MIP-start — fixing only the part you
know (e.g. the combinatorial core of a routing/assignment model) and letting the
solver work out the rest (indicators, flows, continuous bookkeeping).

SCIP supports this via **partial solutions**, where unset variables are `UNKNOWN`
and are filled in by the `completesol` primal heuristic at the start of solving.
This PR exposes them.

## API

- **`Model::<ProblemCreated>::create_partial_sol()`** — create a partial solution
  (`SCIPcreatePartialSol`); unset variables are `UNKNOWN`. Restricted to the
  problem stage, matching SCIP's precondition.
- **`Solution::is_partial()`** — query whether a solution is partial.
- **`add_sol`** now detects partial solutions and routes them to `SCIPaddSolFree`
  (they can't be checked / `SCIPtrySol`'d, so they are added for completion).

```rust
let partial = model.create_partial_sol();
partial.set_val(&x1, 1.0);   // fix only what you know; the rest stays UNKNOWN
model.add_sol(partial)?;      // completesol fills in the rest during solve
```

## Tests

Adds `model::tests::create_partial_sol`: a two-variable covering model seeded with
only one variable fixed (the other left `UNKNOWN`), verifying `is_partial()` and
that the solve reaches the optimum.
